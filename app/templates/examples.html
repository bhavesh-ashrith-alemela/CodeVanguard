level_1_sqli.py CRITICAL BOSS

Level 1: SQL Injection (SQLi)

Uses raw string formatting to inject parameter inputs directly into an SQL execution cursor, allowing arbitrary database extraction. 

import sqlite3

def authenticate(user, password):
    # VULNERABLE: Direct formatting
    conn = sqlite3.connect("prod.db")
    cursor = conn.cursor()
    query = "SELECT * FROM admin WHERE user='%s' AND pass='%s'" % (user, password)
    cursor.execute(query)
    return cursor.fetchone()
level_2_command.py CRITICAL BOSS

Level 2: Command Injection

Runs terminal operations using `subprocess.Popen` with `shell=True` and raw string inputs, exposing the server to command injection. 

import subprocess

def run_diagnostic(ip_address):
    # VULNERABLE: shell=True exposes shell command injections
    cmd = f"ping -c 4 {ip_address}"
    status = subprocess.Popen(
        cmd, 
        shell=True, 
        stdout=subprocess.PIPE
    )
    return status.communicate()

level_4_sandbox.py HIGH THREAT

Level 4: Crypto & Deserialization

Employs weak MD5 hashing and dangerous `pickle.loads` deserialization, which opens access for Remote Code Execution. 

import hashlib
import pickle

def compute_insecure_md5(payload):
    # VULNERABLE: md5 is mathematically broken
    h = hashlib.md5(payload.encode())
    return h.hexdigest()

def unpack_session_data(pickled_data):
    # VULNERABLE: pickle.loads enables arbitrary code execution
    return pickle.loads(pickled_data)
